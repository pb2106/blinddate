# Deploying the IdeaSpace Backend on Railway

This guide walks you through deploying the Express + SQLite backend of the Startup Idea Validator to Railway, a modern and easy-to-use cloud platform.

## Prerequisites
- A GitHub account.
- Your project code pushed to a GitHub repository.

## Step 1: Sign Up / Log In to Railway
1. Go to [Railway.app](https://railway.app/).
2. Click **Login** and authenticate using your GitHub account.

## Step 2: Create a New Project
1. From the Railway dashboard, click the **New Project** button.
2. Select **Deploy from GitHub repo**.
3. Choose the repository that contains your Startup Idea Validator code.
4. Click **Deploy Now**.
   
*Railway will automatically detect your `package.json`, install dependencies (`npm install`), and run the start command (`npm run start` -> `node server.cjs`).*

## Step 3: Add Environment Variables
To ensure your backend handles authentication securely, you must specify a secret key.
1. Click on the newly created deployment card in your Railway project dashboard.
2. Go to the **Variables** tab.
3. Click **New Variable** and add:
   - **VARIABLE NAME:** `SECRET_KEY`
   - **VALUE:** Pick a long, random string (e.g., `my_super_secret_production_key_12345`).
   
*(Note: You do NOT need to set the `PORT` variable. Railway automatically injects the `PORT` your server needs to run on the internet).*

## Step 4: Generate a Public Domain
1. In the same project card view, click on the **Settings** tab.
2. Scroll down until you see the **Domains** section.
3. Click **Generate Domain**.
4. Railway will create a public URL for your assigned server (e.g., `https://your-project.up.railway.app`).

## Step 5: Connect Your Frontend
1. Copy the public domain URL generated by Railway.
2. When deploying your frontend application (e.g., to Netlify or Vercel), add an Environment Variable natively in the frontend platform's dashboard:
   - **Key:** `VITE_API_URL`
   - **Value:** `https://your-project.up.railway.app/api` *(Make sure to append `/api` to the end!)*

Your full-stack application should now be securely deployed, utilizing your robust API backed by SQLite persistent storage!
